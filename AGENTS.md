## 1. TL;DR

- **What it is.** `flag-quiz` is a kid-friendly Progressive Web App (PWA) that
  teaches country flags to young kids (~5+). **Four** difficulty levels,
  bilingual English / Chinese, offline-capable, no backend, no accounts,
  progress in `localStorage`.
- **Stack.** React 19 + TypeScript + Vite 8 + Tailwind 3 + Zustand 5 +
  React Router 7 + framer-motion + `flag-icons` + `vite-plugin-pwa`.
- **Deployed at.** https://jiliny.github.io/flag-quiz/ ŌĆö auto-deployed from
  `main` via GitHub Actions (see `.github/workflows/deploy.yml`).
- **Repo.** https://github.com/jiliny/flag-quiz (public, free plan).
- **Node target.** 20+ (CI uses Node 20; works on 22/24).

```bash
npm install     # first time
npm run dev     # http://localhost:5173/flag-quiz/
npm run build   # tsc -b && vite build && copy dist/index.html -> dist/404.html
npm run preview -- --host   # serve dist on LAN for iPad/phone testing
npm run lint
```

> The `/flag-quiz/` base path is intentional ŌĆö it matches the GitHub Pages
> path. See [┬¦10 Deployment](#10-deployment--github-pages-specifics) before
> renaming the repo.

## 3. Data model

The single source of truth for game content is `src/data/countries.ts`:

```ts
interface Country {
  code: string;                     // ISO-3166 alpha-2, lowercase, e.g. 'us'
  name: { en: string; zh: string };
  hints: { en: string[]; zh: string[] };  // exactly 3 short kid-friendly hints
  pools: { easy: boolean; fill: boolean; medium: boolean; hard: boolean };
}
```

Two exports:

- `CURATED` ŌĆö 50 distinctive, kid-famous countries (default pool).
- `EXTRA` ŌĆö ~60 more, opt-in via Settings ŌåÆ "Include all countries".

Helpers: `getActivePool(expanded)`, `getCountryByCode(code)`,
`nameOf(c, lang)`, `hintsOf(c, lang)`. All countries currently have all three
pool flags = `true`; the `pools` shape exists so we can later restrict a
country to easier levels if its name is too long/hard.

Flag SVGs come from the `flag-icons` package (Tailwind class `fi-${code}`,
imported once in `src/index.css`). Vite only bundles flags actually referenced.

## 4. State management ŌĆö `src/store/gameStore.ts`

A single Zustand store, persisted via `zustand/middleware/persist` to
`localStorage` key **`flag-quiz-state-v1`** (currently `version: 3`).

```ts
{
  mastered: { easy: string[], fill: string[], medium: string[], hard: string[] }, // ISO codes
  settings: {
    lang: 'en' | 'zh',
    sound: boolean,
    expandedPool: boolean,
    testingMode: boolean   // unlock at TESTING_UNLOCK_COUNT instead of full pool
  },
  recentlySeen: string[],  // rolling window of last 40 codes, any level
  coins: number,           // gold coin currency earned by getting flags correct
  unlockedStamps: string[], // unlocked stamp emoji codes, default ['ŌŁÉ']
  countryStamps: Record<string, string> // maps country ISO code to active stamp emoji
}
```

Alongside the state, a `STAMP_CATALOG` constant is exported:
```ts
interface Stamp {
  emoji: string;
  cost: number;
  name: { en: string; zh: string };
}
```
Includes items like Star (`ŌŁÉ`, 0), Balloon (`­¤Äł`, 5), Pizza (`­¤ŹĢ`, 10), Lion (`­¤”ü`, 15), Panda (`­¤É╝`, 15), T-Rex (`­¤”¢`, 20), Whale (`­¤É│`, 25), Unicorn (`­¤”ä`, 25), Race Car (`­¤ÅÄ’ĖÅ`, 30), Rocket (`­¤ÜĆ`, 30), Crown (`­¤ææ`, 50).

Actions: `markCorrect(level, code)`, `markSeen(code)`, `setLang`, `setSound`,
`setExpandedPool`, `setTestingMode`, `buyStamp(stamp, cost)`, `setCountryStamp(code, stamp)`, `reset`.

**Coin Earnings:**
When `markCorrect(level, code)` is called:
- If a flag is already mastered, it awards a `1` practice/review coin.
- If a flag is a new mastery:
  - Easy: `+1` coin
  - Fill: `+2` coins
  - Medium: `+3` coins
  - Hard: `+5` coins

Helpers exported alongside the store (pure functions over a snapshot):

- `poolFor(level, expanded)` ŌĆö filters the active pool by `c.pools[level]`.
- `previousLevel(level)` ŌĆö the prior level in the chain (Easy ŌåÆ Fill ŌåÆ
  Medium ŌåÆ Hard). Returns `null` for Easy.
- `isLevelUnlocked(level, mastered, expanded, testingMode = false)` ŌĆö Easy is
  always unlocked; each later level unlocks when **every** country in the
  previous level's pool has been mastered. With `testingMode === true`, the
  threshold drops to `min(TESTING_UNLOCK_COUNT, prevPool.length)` countries
  (default `TESTING_UNLOCK_COUNT = 5`).
- `levelProgress(level, mastered, expanded) -> { done, total }` ŌĆö `done`
  intersects the persisted set with the current pool (handles users
  toggling `expandedPool` mid-progress).

### Store schema migrations

Current version is **`3`**. A `migrate` function in `persist`'s config handles:
- **v1 ŌåÆ v2**: preserves `easy`/`medium`/`hard` mastered arrays, adds empty `fill` array, defaults `testingMode` to `false`.
- **v2 ŌåÆ v3**: introduces `coins` (defaults to `0`), `unlockedStamps` (defaults to `['ŌŁÉ']`), and `countryStamps` (defaults to `{}`).

> **Bumping the store schema:** bump `version`, add another `if (version < N)`
> branch in `migrate`, and write **defensive** code (always assume each
> previous-version field may be missing). Don't silently change the shape ŌĆö
> old users without a migration will lose progress.

## 5. How rounds work (shared mechanics)

All three rounds follow the same flow and live in `src/rounds/`:

1. **Pick** an unmastered country via `pickNext(pool, mastered[level], recentlySeen)`.
   Preference order: unmastered & not recently seen ŌåÆ unmastered ŌåÆ anything.
2. **Render** via `<RoundShell>` (shared header with back button, progress,
   sound toggle, plus flag area and input area). Use `<FlagCard size="hero|lg|md|sm">`.
3. **Receive input** ŌåÆ check via `isCorrect(guess, truth, lang)` (which
   normalizes case + strips non-letters for `en`, strips whitespace for `zh`).
4. **On correct**: `audio.correct()` ŌåÆ confetti via `celebrateAt(el)` ŌåÆ
   `markCorrect(level, code)` (adds coins!) ŌåÆ 1.2ŌĆō1.3 s success banner ŌåÆ `advance()` to next.
5. **On wrong**: `audio.wrong()` ŌåÆ increment `wrongCount` ŌåÆ input clears ŌåÆ
   `<HintCard>` reveals `hints[lang][(wrongCount - 1) % 3]`. **No answer is
   ever revealed.** The kid keeps trying.

### Hint cycling ŌĆö the contract

- Each country has **3 hints per language** (enforced by data convention).
- The Nth wrong attempt shows hint index `(N - 1) % hints.length`.
- The card re-mounts on every attempt (`key={attemptIndex + text}` in
  `HintCard.tsx`) so the slide-in animation re-fires.
- Behavior is identical across Easy / Medium / Hard. If you add a new level,
  use the same `useState<number>(0)` counter + `wrongCount > 0 ? hints[(wrongCount-1) % len] : null` pattern.

### Per-level specifics

| Round  | Input                                    | Wrong UX                                                                |
| ------ | ---------------------------------------- | ----------------------------------------------------------------------- |
| Easy   | 1 correct + 3 distractor name tiles      | Tapped tile dims + Ō£Ś; kid taps a different tile (no clear needed)       |
| Fill   | Mostly-revealed name + 1ŌĆō4 blanks; QWERTY (en) or 18-cell char grid (zh) fills the blanks | Blank slots clear after 700 ms; keyboard re-enables; hint advances |
| Medium | Letter/char tiles fill ordered slots     | Slots clear after 700 ms; tiles re-enable; hint advances                |
| Hard   | On-screen keyboard fills slots           | Slots clear after 800 ms; keyboard re-enables; hint advances            |

The Medium tile bank size is `max(12, targetChars + 4)` for `en` and
`max(12, chars + 8)` for `zh`. The Hard `zh` keyboard uses a 36-cell char
grid built from the answer chars + decoys sampled from other countries'
Chinese names (see `buildHardSetup` in `HardRound.tsx`).

### Fill round ŌĆö blank rules (see `blanksForLength` and `pickBlankIndices`
in `src/lib/quiz.ts`)

For a name with `N` blank-able positions (Latin letters for `en`, CJK chars
for `zh`):

| Lang | N range | Blanks |
| ---- | ------- | ------ |
| en   | 1ŌĆō4     | 1      |
| en   | 5ŌĆō7     | 2      |
| en   | 8ŌĆō11    | 3      |
| en   | 12+     | 4      |
| zh   | 1ŌĆō4     | 1      |
| zh   | 5+      | 2      |

Position-selection rules (in `pickBlankIndices`):
- **English: never blank the first letter** ŌĆö kids need a visual anchor.
  (This rule does **not** apply to Chinese; for `zh` we want even the leading
  character to be in play, so `_ÕøĮ` is a valid puzzle and exercises the kid's
  knowledge of the meaningful first character.)
- Never blank spaces or punctuation (`predicate` controls what's eligible).
- **Greedy non-adjacent placement** ŌĆö shuffle candidates, then accept each
  only if no already-picked blank is within 1 position. Fall back to closer
  picks only if the string is too dense to satisfy the constraint.
- Layout is fresh per round but stable across retries (state is in
  `useState`'s init for the round; not regenerated on wrong answer).

## 6. i18n

- Dicts live in `src/i18n/en.ts` and `src/i18n/zh.ts`. The English dict is
  the canonical shape; `zh.ts` must mirror it.
- `useT()` returns a `(key, params?) => string` with nested-key lookup
  (`"levels.easy"`) and `{name}` interpolation.
- `useLang()` returns the current `'en' | 'zh'`.
- Country names/hints are in `countries.ts`, **not** the dicts.

To add a new language:

1. Add `xx` to the `Lang` union in `src/data/countries.ts`.
2. Create `src/i18n/xx.ts` mirroring `en.ts`.
3. Register it in `src/i18n/index.ts` (`DICTS`).
4. Add `name.xx` and `hints.xx` to **every** country (~109 entries ŌĆö script it).
5. Add a flag emoji + label to the language pill in `src/pages/Home.tsx`
   and the radio in `src/pages/Settings.tsx`.
6. If the script is non-Latin and non-CJK, the Hard `Keyboard` and Medium
   tile builder may need new branches (see `buildMediumTiles` and
   `buildHardSetup`'s `lang === 'en' ? ... : ...` switches).

## 7. Styling system

Tailwind tokens are defined in `tailwind.config.js`:

- **Color families.** `sky`, `mint`, `sun`, `candy` each in `soft/main/deep`.
  Plus `ink` (near-black text) and `cream` (page bg).
- **Shadows.** `shadow-sticker` (polaroid feel), `shadow-button` (chunky 3D
  bottom edge), `shadow-keycap` (subtler 3D for keyboard).
- **Animations.** `animate-wobble` (continuous flag tilt), `animate-shake`
  (single-shot wrong feedback), `animate-pop` (entrance), `animate-floaty`
  (Globie idle bob). All durations chosen to feel non-frantic for 5-year-olds.
- **Radii.** `rounded-xl2` = 28px (large kid-friendly), `rounded-pill`.
- **Font.** Fredoka via `@fontsource/fredoka` (imported in `index.css`,
  bundled ŌĆö works offline).

### The flag aspect-ratio rule & height-responsive scaling

`FlagCard` uses **fixed pixel dimensions per size variant** (`sm/md/lg/hero`)
with `maxWidth: '100%'` for small-screen scaling and
`backgroundSize: 'contain'`. The result is a hard 4:3 frame that letterboxes
non-4:3 flags (e.g. Nepal) instead of stretching. **Do not** change to
percentage widths or `cover` ŌĆö flags will distort with window resize, which
was the explicit complaint we fixed.

If you add a new size, add it to `sizeMap` in `src/components/FlagCard.tsx`,
including its `padPx` for the outer polaroid wrapper.

**Height-based responsive scaling (Pixel 10 Pro XL layout fix):**
To ensure the layout fits perfectly on devices of all shapes and sizes (including tall viewports like Google Pixel 10 Pro XL, small viewports like iPhone SE, or wide viewports like tablets) without causing double scrollbars or button cutoffs:
1. **Isolated Scroll:** `RoundShell` uses a rigid `h-dvh` + `overflow-hidden` container. The header is sticky, and the main play area is isolated via `flex-1 min-h-0 overflow-y-auto`.
2. **Aspect Scaling:** `src/index.css` defines `.flag-card-responsive` scaling via `@media (max-height: 700px)` and `@media (max-height: 600px)`. It automatically shrinks the maximum width and height of the polaroid card (e.g., from `320px` to `260px` or `200px`) and scales the inner flag image.
3. **Compact Buttons:** Button dimensions are responsive. Standard buttons in rounds like `EasyRound` drop from `py-5 text-3xl min-h-[5rem]` on large screens to compact `py-3 text-xl min-h-[3.25rem] gap-2` on mobile viewports.

### `no-select` and `safe-pt`/`safe-pb`

These are custom utilities defined in `src/index.css`:

- `no-select` ŌĆö disable text selection on tap targets (kids long-press constantly).
- `safe-pt` / `safe-pb` ŌĆö `env(safe-area-inset-*)` padding for iPhone notches
  and Android nav bars when installed as a PWA.

## 8. Sound ŌĆö `src/lib/audio.ts`

Pure WebAudio synthesis, **zero** audio files. Five SFX: `tap`, `correct`,
`wrong`, `unlock`, `stamp`. The first user gesture in each round calls
`audio.unlockUserGesture()` to satisfy iOS autoplay policy.

The `sound` toggle is read in `App.tsx` and propagated via
`setSoundEnabled(v)` so individual `audio.*` calls become no-ops when off.

## 9. PWA / offline

Configured by `vite-plugin-pwa` in `vite.config.ts`:

- `registerType: 'autoUpdate'` ŌĆö new SW activates on next visit; no prompt.
- `workbox.globPatterns: ['**/*.{js,css,html,svg,png,woff2}']` ŌĆö flags,
  Fredoka font, and the app shell are all pre-cached.
- `maximumFileSizeToCacheInBytes: 5 MiB` ŌĆö needed because some bundled flag
  SVGs are big when many are included.
- Manifest `start_url` and `scope` are both `/flag-quiz/` ŌĆö match `BASE`.

Icons are generated from `public/favicon.svg` via:

```bash
npm run generate-pwa-assets
```

This rewrites the four PNGs (`pwa-64`, `pwa-192`, `pwa-512`,
`maskable-icon-512x512`) and `apple-touch-icon-180x180.png`. Commit them.

> If you change the SVG, also re-verify on iOS Safari "Add to Home Screen" ŌĆö
> Apple's caching is aggressive and a stale icon may persist.

## 10. Deployment ŌĆö GitHub Pages specifics

The deploy pipeline is dead simple:

1. Push to `main`.
2. `.github/workflows/deploy.yml`: `actions/checkout@v4` ŌåÆ `setup-node@v4`
   (Node 20, npm cache) ŌåÆ `npm ci` ŌåÆ `npm run build` ŌåÆ
   `upload-pages-artifact@v3` ŌåÆ `deploy-pages@v4`.
3. **One-time** repo setup that's already done: `gh api -X POST
   repos/jiliny/flag-quiz/pages -f build_type=workflow`.

### Don't break these invariants

- **`base: '/flag-quiz/'`** in `vite.config.ts` MUST equal `/<repo-name>/`.
  If you rename the repo, update `base`, then update the manifest
  `start_url`/`scope` (which use the same `BASE` constant), then re-deploy.
- **`public/.nojekyll`** is required so Pages doesn't try to Jekyll-process
  the `_*` files some bundlers emit.
- **`scripts/spa-404.mjs`** copies `dist/index.html` to `dist/404.html`.
  GitHub Pages serves `404.html` for any path that doesn't match a static
  file, which means React Router takes over for deep links like
  `/flag-quiz/game/easy`. Don't remove this script.
- The repo is **public** because GitHub Pages requires Pro for private repos
  and we're on the free plan.

### Custom domain

Drop a `CNAME` file in `public/` with the apex/subdomain. Then in
`vite.config.ts` change `const BASE = '/flag-quiz/'` to `const BASE = '/'`,
and reconfigure DNS per
[GitHub's instructions](https://docs.github.com/en/pages/configuring-a-custom-domain-for-your-github-pages-site).
Also delete the `404.html` workaround if you switch to a host that supports
SPA fallbacks natively.

### Watching a deploy

gh run watch <run-id> --exit-status

## 11. Common tasks (HOWTOs)

### Add a new country

1. Pick a 2-letter ISO code (lowercase). Verify it exists at
   https://github.com/lipis/flag-icons/tree/main/flags/4x3.
2. Decide: curated or extra? If kid-famous and the flag is distinctive ŌåÆ
   `CURATED`. Otherwise ŌåÆ `EXTRA`.
3. Add the entry to `src/data/countries.ts`. **Three hints in each language.**
   Hint conventions:
   - Short (Ōēż 7 words English / Ōēż 8 chars Chinese ideally).
   - Concrete and visual: animals, food, landmarks, sports.
   - No tricky geography or politics.
   - Don't put the country name in the hint (the kid is trying to guess it).
4. Run `npm run dev` and play through Easy/Medium/Hard with that flag.

### Change which countries are in the default pool

Move entries between `CURATED` and `EXTRA`. Don't change the `code` ŌĆö that's
the persisted ID in users' `localStorage`.

### Adjust difficulty

- **Easier Easy:** lower the option count from `buildEasyOptions(pool, answer, 4)`.
- **Easier Fill:** tweak `blanksForLength` in `lib/quiz.ts` ŌĆö drop the count
  per length bucket (e.g. `1ŌĆō7 ŌåÆ 1` instead of `5ŌĆō7 ŌåÆ 2`).
- **Harder Fill:** raise `blanksForLength`, or relax the
  "no first-letter blank for en" rule in `pickBlankIndices`.
- **Harder Medium:** increase the decoy tile count in `buildSlotting`
  (`totalTiles = Math.max(12, targetCount + N)` ŌĆö bump `N`).
- **Different unlock rule:** edit `isLevelUnlocked` in `gameStore.ts`. Today
  it's "100% of previous pool" (or `TESTING_UNLOCK_COUNT` when testing mode
  is on). You could make the production threshold "80%" by changing the
  `prevPool.every(...)` branch.
- **Different repetition:** edit `pickNext` in `lib/quiz.ts`. The
  `recentlySeen` window is 40 (`markSeen`); shrink for more variety on small pools.

### Use testing mode while developing

Settings ŌåÆ "Testing mode" (a ­¤¦¬ toggle). When on, each level unlocks after
just **5** countries instead of the whole pool. The threshold is the
constant `TESTING_UNLOCK_COUNT` in `gameStore.ts` ŌĆö change it there.

This is purely a developer/QA convenience; it's wired to localStorage like
any other setting, so it persists per device. Leave it OFF for the real
play experience.

### Stamps & Passport Shop Customization

Stamps are defined in the `STAMP_CATALOG` array in `src/store/gameStore.ts`.
To add or modify stamps:
1. Add/edit the stamp object with its `emoji`, `cost` (in coins), and bilingual `name` (`en` / `zh`).
2. Custom assets/symbols can be added directly via emoji.
3. Translation keys like `stampThisCountry` or `unlockStamp` are defined in `src/i18n/en.ts` and `src/i18n/zh.ts`.

## 12. Conventions

- **Imports.** Use relative imports inside `src/` (no path aliases configured).
- **Types.** Strict TS. Prefer `interface` for shapes, `type` for unions/aliases.
- **Components.** Function components only, named exports (`export function X`).
  Pages and rounds are the exception that re-export under their own name.
- **Animations.** Prefer `framer-motion` for state-driven motion;
  Tailwind keyframes for purely decorative loops (wobble, floaty).
- **Side effects in render.** `useEffect(() => markSeen(code), [code])`
  is the standard pattern; the round picks a country in `useState` init and
  marks it seen via effect ŌĆö don't move this into render.
- **No backend.** Don't add `fetch`/network calls. Everything must work offline.
  If you need a remote dataset, bake it at build time into `src/data/`.
- **No bundle bloat.** Don't import individual icon libraries (`lucide`,
  `heroicons`, etc.) ŌĆö we use inline SVG / emoji for icons. Adding a new
  large dep needs a strong justification.

## 13. Known limitations / future work

- **Hardcoded 3 hints.** The code assumes `hints.en.length === 3 === hints.zh.length`.
  Generalization would require auditing the `% hints.length` math in all
  three rounds (it'd still work; just no longer "exactly 3 unique tries
  before repeating").
- **Repetition.** With `expandedPool: false` (50 countries) and
  `recentlySeen` of size 40, a kid will see near-duplicates after mastering
  most of the pool. Consider bumping the window or weighting unmastered more
  aggressively in `pickNext`.
- **No per-kid profiles.** Single device = single progress. If we want
  siblings or "guest mode", the store needs a profile dimension.
- **No tests.** This is a small app and we lean on type-checking + manual
  play-through. If you add a test runner, `lib/quiz.ts` (`pickNext`,
  `buildEasyOptions`, `buildMediumTiles`, `isCorrect`, `answerSlotCount`)
  is the natural starting point ŌĆö all pure functions.
- **No analytics.** Intentional ŌĆö kids' app, no tracking.
- **CI uses Node 20.** GitHub announced auto-upgrade to Node 24 on
  2026-06-02. Our pinned actions (`checkout@v4`, `setup-node@v4`,
  `upload-pages-artifact@v3`, `deploy-pages@v4`) all support Node 24, so
  it should be a no-op. If a deploy starts warning loudly, opt in early by
  adding `env: FORCE_JAVASCRIPT_ACTIONS_TO_NODE24: 'true'` at the workflow
  level.

- **Branch.** `main` only (no `dev`). Pushes to `main` deploy.
